# FitLos

Интеллектуальный ассистент для фитнес-клуба «Лос Айленд». Система автоматизирует обработку клиентских запросов из мессенджера MAX, анализирует диалоги с помощью GigaChat API и технологии RAG, прогнозирует риск оттока клиентов (Churn Rate) и предоставляет администраторам дашборд с готовыми рекомендациями и черновиками ответов.

**Статус:** дипломный проект (ВКР)  
**Университет:** РЭУ им. Г. В. Плеханова, ВШКМиС, кафедра информатики

## Что умеет система

- **Авторизация и управление пользователями** — регистрация, вход, сброс пароля с JWT-токенами и хэшированием паролей.
- **Дашборд администратора** — три вкладки:
  - *Клиенты* — таблица с ФИО, телефоном, риском оттока (churn_risk), датой последнего визита. Кнопка синхронизации с Yclients.
  - *Чаты* — аккордеон с переписками из мессенджера MAX, сгруппированными по клиентам. Кнопка «Анализировать диалог» для каждого чата.
  - *Аналитика* — метрики (время ответа, конверсия, сегментация по риску и активности), список клиентов с высоким риском оттока (>60%) с персонализированными действиями.
- **Интеграция с GigaChat API** — асинхронные запросы к LLM, генерация структурированных JSON-ответов (summary, sentiment, churn_probability, draft_response).
- **RAG-механизм** — поиск релевантных фрагментов в базе знаний клуба (прайсы, скрипты, правила) и подстановка их в системный промпт.
- **Синхронизация с Yclients** — загрузка клиентов через API, расчёт churn_risk по формуле: `0.5×recency + 0.3×frequency + 0.2×monetary`.
- **База данных PostgreSQL** — таблицы users, clients, chat_logs (модели SQLAlchemy).

## Как запустить локально

### 1. Клонируйте репозиторий

```bash
git clone https://github.com/I1ne/fitlos-analytics.git
cd fitlos-analytics
```

### 2. Создайте и активируйте виртуальное окружение

```bash
python -m venv venv
source venv/bin/activate        # для Linux/macOS
# venv\Scripts\activate         # для Windows
```

### 3. Установите зависимости

```bash
pip install -r requirements.txt
```

### 4. Настройте PostgreSQL

Убедитесь, что PostgreSQL запущен, и создайте базу данных:

```bash
createdb fitness_db
```

### 5. Создайте файл `.env` в корне проекта

```env
# GigaChat API
GIGACHAT_CREDENTIALS=ваш_токен_авторизации
GIGACHAT_SCOPE=GIGACHAT_API_PERS
GIGACHAT_MODEL=GigaChat

# Yclients API (опционально — для синхронизации)
YCLIENTS_API_URL=https://api.yclients.com/api/v1
YCLIENTS_PARTNER_TOKEN=ваш_партнёрский_токен
YCLIENTS_USER_TOKEN=ваш_пользовательский_токен

# База данных
DATABASE_URL=postgresql://user:password@localhost/fitness_db

# JWT-секрет
SECRET_KEY=your-secret-key-change-it
```

### 6. Запустите сервер

```bash
uvicorn app:app --reload
```

### 7. Откройте в браузере

Перейдите по адресу [http://127.0.0.1:8000](http://127.0.0.1:8000).  
Зарегистрируйтесь и войдите в систему.
